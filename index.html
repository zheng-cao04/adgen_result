AdGen

Development snapshot

What brief.json is

brief.json is the pipeline input contract: a compact product and ad request supplied by the user.

brief.json fields
product.name
The product or service being advertised.
product.category
Optional planner hint such as beauty, food, electronics, software tool, fashion, or furniture.
product.selling_points
Benefits, constraints, proof points, or required demo actions the ad should preserve.
product.image_path
Optional product reference image for identity. Direct route sends it to image-edit/ref2img to create a full-scene start frame; keyframe route uses it for key-image stages.
language
Output language for planner text and generated ad prompts.
duration_seconds
Target total video length for the generated ad.
execution_route
direct asks the planner for one single-shot full-video plan; keyframe asks for hook/ad shots with key-image prompts.

High-level flow

From that brief, AdGen builds inspectable JSON artifacts at each step. The current control plane has two video routes: direct builds one full-length request and, when a product image is present, first turns that image into a generated start frame; keyframe uses start images for hook/ad image-to-video jobs. After generation, the critic can accept the clip or trigger one GPT prompt rewrite for a retry.

1. brief.json The user-facing input contract: product details, constraints, references, and duration.
2. ad_blueprint.json The GPT planner creates strategy, asset memory, and a route-specific execution layer.
3. route selection direct may create one direct start frame. keyframe consumes hook/ad start_key_image_path values.
4. video jobs Build provider-neutral requests: one full job for direct, or hook/ad jobs for keyframe.
5. video generation The HTTP backend prints /version, generates mp4 outputs, and concatenates hook/ad clips when present.
6. critic + retry Qwen-VL returns pass, summary, and failure_reasons. Failed clips can get a GPT-rewritten prompt.

Stage 1: brief.json -> ad_blueprint.json

The planner stage converts a compact user brief into ad_blueprint.json. The output shape now follows execution_route. Direct route asks GPT for one unified single-shot plan. Keyframe route asks GPT for hook/ad shots and key-image prompts. This stage still does not generate media.

Stage 2A: direct route -> direct start frame -> video_jobs.json

The one-shot path skips hook/ad key-image generation. When product.image_path is present, AdGen first uses image-edit/ref2img to place the real product into one full-scene start frame. The job builder then reads the updated direct single_shot plan and emits one full-length request with metadata.role = "full".

Stage 4: video_jobs.json -> generated mp4

The generation stage executes each request in video_jobs.json through the HTTP video backend. The client prints the server /version response before generation so a stale remote server is visible immediately. Direct jobs produce one 01_full.mp4; keyframe hook/ad jobs can be concatenated into combined.mp4.

Command

export VIDEO_API_BASE="${VIDEO_API_BASE:-http://127.0.0.1:58083}"

python scripts/generate_video.py "${RUN_DIR}/video_jobs.json" \
  --backend http \
  --api-base "${VIDEO_API_BASE}" \
  --timeout-seconds 7200

{

Stage 5: generated mp4 -> critic_report.json

The critic stage evaluates generated clips against the compact checklist stored in video_jobs.json. It uses a local Qwen-VL video critic to produce a structured pass/fail report. The current schema intentionally removes numeric scores and long evidence fields; it keeps only pass, summary, and failure_reasons per clip.

Command

python scripts/critic_video.py "${RUN_DIR}/video_jobs.json" \
  --model-path models/Qwen3-VL-8B-Instruct \
  --only-role full \
  --fps 1 \
  --max-pixels 230400 \
  --output "${RUN_DIR}/critic_report.json"

Input expectation shape

{
  "metadata": {
    "role": "full",
    "critic_expectation": {
      "purpose": "Judge whether the raw video is a plausible product demo for Portable LED Lamp.",
      "hard_requirements": [
        "single continuous raw video shot with no obvious cut, montage, split screen, or scene change",
        "main product demo action: the lamp is placed, turned on, and the warm light is visible"
      ],
      "soft_context": [
        "Do not fail only because the pain point, emotional transition, or every optional prop is incomplete."
      ],
      "must_avoid": [
        "unrequested subtitles, captions, UI, watermarks, fake logos, background text, or random letters"
      ]
    }
  }
}

Output: critic_report.json

{
  "pass": false,
  "clips": [
    {
      "job_id": "portable_lamp_one_take_v1_full",
      "role": "full",
      "clip_path": "runs/electronics_.../video/01_full.mp4",
      "pass": false,
      "summary": "A hand places a small lamp on a table, but the lamp does not visibly light the wall.",
      "failure_reasons": [
        "The main product proof is missing: the warm light is not visible on the tabletop or wall."
      ]
    }
  ]
}
Pass Accept generated clips and end the run.
Fail Use failure_reasons with the original prompt as input to GPT prompt rewriting.
Retry Regenerate the failed role from video_jobs_retry_01.json, then run the critic again.

A separate key-image critic still exists for multi-candidate key-image selection, but it is not part of the direct route. Video critic failures are handled by scripts/retry_video_from_critic.py.

Stage 6: critic fail -> GPT prompt retry

Retry no longer has a rule-based prompt patcher. A failed clip is retried only by calling GPT with the original video prompt and the critic summary/failure reasons. The planner output and any start images stay fixed.

Command

export AZURE_OPENAI_PROMPT_REWRITE_MAX_COMPLETION_TOKENS=8000
export PROMPT_REWRITE_MAX_WORDS=160

python scripts/retry_video_from_critic.py \
  "${RUN_DIR}/video_jobs.json" \
  "${RUN_DIR}/critic_report.json" \
  --only-role full

Output artifacts

{
  "video_jobs_retry_01.json": "same request shape, failed prompt replaced by GPT rewrite",
  "video_retry_plan_01.json": {
    "retry_index": 1,
    "adjusted_requests": [
      {
        "role": "full",
        "job_id": "portable_lamp_one_take_v1_full",
        "retry_output_path": "runs/electronics_.../video/retry_01/01_full.mp4"

  }
}
